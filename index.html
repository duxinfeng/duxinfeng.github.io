怎么判断viewcontroller是Push还是Pop?

一般情况下判断viewcontroller在返回时需要push还是pop都会在.h文件带一个参数进来。如何让容器自己判断？

方法一

NSArray* views = [self.navigationController viewControllers];
BOOL variable = [views containsObject: self];
if (variable) {
    [self.navigationController popViewControllerAnimated:YES];
}
else {
    [self dismissViewControllerAnimated:YES completion:^{}];
}

方法二

if (self.navigationController != nil) {
    [self.navigationController popViewControllerAnimated:YES];
}else{
    [self dismissViewControllerAnimated:YES completion:^{}];
}

为何你会一年经验用十年

我在看个人Blog或者搜到一些个人Blog的时候对他们的关于比较感兴趣，然后就是他们的总结、反思、感悟等，我想知道他们在思考什么想什么，也想知道被人是如何介绍自己的。

在http://unclejoke.github.io/business.html 这篇文章里，博主讲讲到学习和招聘的时候让我头脑发麻。特别是在先看到博主在招聘一段说的最近公司一直在招各方面的人才，对于IOS的初级开发者我大致可以分为两类，一是刚培训完成，简历上各种精通,大谈MVC,单例等设计模式之类，而问到实际的问题却什么也不会；另一种是一两年经验，同样是各种精通，而却连一份基础的面试题的一半都不能答对。这让我想起了之前看到的一篇文章：为何你会一年经验用十年里边很刺眼的一句话：“别说你有十年经验，你只是一年经验用了十年而已。” 一年经验能用十年也许不是贬义，但确实是一种悲哀。，当再去看他谈学习的时候，今天在群里与人讨论到了SDWebImage 的实现问题，我提了一个问题，假如让你来写一个图片的缓存处理，你会这么设计？对方的观点是，这么好用的库直接用就好了，干嘛还要去重新写一遍。当然对于这个观点我也是极为赞同的，但是在深入阅读过很多源码之后你会发现你你学到了好多，牛人总有牛人的原因所在，这便是学习。,正好这时我当前正在趋向的一个状态，从编写NSLog(@"Hello word");开始学习iOS差不多满四年了,这时目前比较恐惧的一块，中间尝试过去学习PHP、Python、JS甚至是Swift都是半途而废，没有完整的概念和流程。每天早上到公司刷刷微博，看看关注的大牛门转发的文章，去github和CocoaControl上start开源的项目，所以基本基本上多iOS常用或不常用的库都有所了解，也就是眼熟，遇到一些功能问题的时候第一步想的就是第三方。中间尝试去扒一扒开源项目实现原理，学习学习源码，依旧中途夭折，也就是说还钻不进去了。 就想在另一个另外一个地方看到这样一段调侃的话廖雪峰的git教程中，刚开始的部分的浏览量有38万，但是最后一页教程的网页的浏览量是1万多。让很多人躺枪，我也是这很多人里面的。

为何你会一年经验用十年原文链接是， http://davidzhang33.blog.51cto.com/3095817/1313940，博主在谈学习的时候写了四篇。
