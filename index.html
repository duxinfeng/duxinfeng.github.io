使用支付宝SDK遇到的坑

下载SDK

有可能是我搜索姿势不对，但是在网上看到很童鞋吐槽支付宝SDK藏得太深很不好找。

和很多童鞋一样在Google/Baidu搜索框输入支付宝SDK,然后第一条结果https://openhome.alipay.com/doc/docIndex.htm?url=https://openhome.alipay.com/doc/viewKbDoc.htm?key=236698_261849&type=info(Baidu要忽略掉前面推广部分的第一条)，然后点到SDK下载跳到：

纳尼？版本2.0最新更新日期2013-10-10，好歹也是大公司，不科学啊,果然下载下来的SDK也不正确。。。

然后在网上扒扒扒。。。

找到原因

原来关键词输入错误，应该搜索支付宝开放平台（http://doc.open.alipay.com/doc2/alipayDocIndex.htm。

或者在搜索支付宝SDK出来页面的时候,点击导航栏跳转一下就好。

下载坑

不知道是什么原因，在网页上下载完，但是一直是这个状态,应该是解压打不开。最后用的迅雷。

支付宝支付问题

文档里建议：“出于安全考虑，请商户尽量把私钥保存在服务端，在服务端进行签名验签。”，所以客户单在本地拿到订单信息之后进行字符串拼接，然后给丢到支付宝SDK中,等待处理结果。

[[AlipaySDK defaultService] payOrder:orderString fromScheme:appScheme callback:^(NSDictionary *resultDic) {
       NSLog(@"reslut = %@",resultDic);
   }];

注意

  • 如果出现6001错误（用户中途取消），请检查订单在做拼接时候是否符合规则。参数需要有“双引号”包起来，因为这个也排查好一会儿才找到错误原因。

  • 运行Demo前，需要作为商户到 https://b.alipay.com 进行签约与审核，才能拿到 partner、seller、privateKey 信息，demo中参数配置在APViewController.m文件里。

Git忽略UserInterfaceState.xcuserstate文件

关于UserInterfaceState.xcuserstate文件

路径：ProjectFolder.xcodeproj/project.xcworkspace/xcuserdata/myUserName.xcuserdatad/UserInterfaceState.xcuserstate

每次打开工程，动一动Xcode UserInterfaceState.xcuserstate就会出现变动。

解决办法

因为我们团队使用SourceTree进行代码管理，选中UserInterfaceState.xcuserstate右键出现菜单，然后选中停止追踪，然后进行commit。如果是对人协作开发，团队其他人员也需要执行相同操作。

还有其他解决方案，原理一样

git rm --cached ProjectFolder.xcodeproj/project.xcworkspace/xcuserdata/myUserName.xcuserdatad/UserInterfaceState.xcuserstate

git commit -m "Removed file that shouldn't be tracked"

原地址：http://stackoverflow.com/questions/6564257/cant-ignore-userinterfacestate-xcuserstate

碎碎念在12月6日

搬家

从去年的五月份的时候说搬家，在公司部分资助一直搬到现在，搬了一年半终于搬离天通苑北，来到公司附近，现在东西全部办过来了，找了一两个星期的房子，明天再回去交接一下，终于心里一个石头可以落下了，不过还的置办生活用品。

Swift开源

本周最重磅的消息就是苹果兑现承诺，终于赶在年底前把Swift开源了，官网https://swift.org,听说当天被手动DDos了，访问不到。github:https://github.com/apple/swift开源当天star过万，截止目前已有19010个star，170人贡献代码。也该准备准备好好学习下Swift了。

关于ASO

周六app被拒，因为在做ASO的时候标题写的有点过，大概五月份苹果就开始严查副标题，虽然可以写255个字，如果你的副标题太长还是会被拒，然后建议你写到关键词或者描述里面，关于ASO标题、副标题、关键词等一些权值在八月份的时候学习了一段时候，然后做了一次分享，PPT:https://docs.google.com/presentation/d/19Jp49NY07U25GQ_PUAQSH7VzChgTboJ5LvZvX4HMuhY/edit?usp=sharing.
因为关键词限制255个字符，偶然得知，关键词可以不隔开，所有的词连在一起，这样被拆分后的关键词覆盖可以超过1200+，不过也有缺点，就是会产生较多的无用关键词，总体来说还是好的。

线上读书活动完成

三周前我参加了一个21天挑战——每天读书半小时第一期的活动，今天正好结束，502人参加，156人挑战失败，第一天的时候超过40人挑战失败，然后挑战完成不仅可以拿回自己的本金，还可以平分失败人得本金，能多得45.22.

从理财角度上分析，我用100元，12天后我得到145.22，这个投资回报率超过45%。不过从收获方面来看自己学到的东西属于自己，谁也拿不走,这样就转的更多了。

谈一点体会：

这21天里我读了《人性的弱点》（100%）、《黑客与画家》（90%），因为书中有的章节自己现在这个水平和精力读起来还是难以理解，所以会显得枯燥，然后读了之后没有任何感觉，这个时候最好的办法的就是换一本容易理解的书去读，比如人物传记。因为每天夜晚到家晚，所以为了完成读书任务，在读起来的时候明显觉得有点赶，不能静心来去体会，也会造成效率差。

从中最大收获就是，发现自己读的书太少，也就是你读的书越多，就会发现自己读的越少，很矛盾，但是那是你的真是体会。在读一本书的时候，你能从这本书中找到下一本或下下本要读的书，就是这么神奇。

当然，书读完完了，还差读后感，需要补上。

2、看到一篇不错的博客

《如何提升你的能力？给年轻程序员的几条建议》,博主建议很中肯，从中提取的几点：

  • 每个程序员都应该经营一款自己的产品，它可以是一款app，一个网站或是一个开源软件。
  • 我觉得每个人都需要一个职场导师，他可以是你第一份工作的导师或是Manager，也可以是你认识的其他前辈。你们需要维系一个非常长期的关系，不止于一家公司，最好贯穿你的整个职业生涯。每当你遇到疑惑时，都可以询求他的建议，我觉得这将是你最宝贵的一笔人脉财富。
  • 在面试一家公司的时候，你要意识到面试是双向的，公司在面试你的同时，你也在面试这家公司。

第一条我正做，第二条比较幸运，冥冥之中存在着那么一个Manager，第三条是我读完之后才能意识到，或者是表达除了我们在面试的时候想法。

The identity used to sign the executable is no longer valid

问题描述

公司一个账号，然后在多台设备使用。

The identity used to sign the executable is no longer valid.

Please verify that your device’s clock is properly set, and that your signing certificate is not expired. (0xE8008018).

Preferences —> Accounts —> View Details
Reset和Download all 都使过，不管用。

包括按照参考资料2.http://pinkstone.co.uk/how-to-fix-the-identity-used-to-sign-the-executable-is-no-longer-valid-error-in-xcode/也无果。

最终解决姿势

还是简单粗暴的从Xcode移除Apple id,然后重新登录就好了。

原话:Remove your apple id from Xcode preference and add it back again. Clean, rebuild and make sure coding signing is correct in project settings. That works for me.
